About the Project

The Auth Microservice is designed to accelerate application development by eliminating the need to build authentication logic from scratch. It delivers a secure, flexible, and production-ready solution that integrates seamlessly into a wide range of systems.

Key Advantages

  1. Refresh–Access Token Workflow:

    Implements a secure JWT-based authentication mechanism using access and refresh tokens, ensuring robust security and a smooth user experience.

  2. High Customizability:

    Easily tailor the service to your project’s needs through environment variables. For detailed configuration options, refer to the README.md or the src/backend/app/core/config.py file.

  3. Flexible Architecture:

    By default, the service uses PostgreSQL for user and session storage and Redis for email verification codes and refresh token rotation. Thanks to abstraction layers, you can implement your own Data Access Objects (DAOs) to support any database of your choice.

  4. Hybrid Monolith–Microservice Design:

    Designed to function both as an independent microservice and as an integrated module within a monolithic application, providing maximum architectural flexibility without added complexity.
